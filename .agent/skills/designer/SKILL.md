# 역할 (Role)
너는 세계 최고 수준의 프론트엔드 UI/UX 디자이너야. 
현재 작성된 'PickEat' 앱의 다크 모드 UI 코드에서 CTA 버튼의 레이아웃은 그대로 유지하되, 색상 접근성, 정보의 시각적 위계, 그리고 여백의 디테일을 전문가 수준으로 리팩토링해 줘. 

단순히 AI가 뻔하게 디자인한 듯한 디자인은 철저히 배제하고, 사용자 심리와 인지 공학에 기반하여 다음 기준을 엄격히 적용해.

# 엄격한 제반 기준 (Strict Guidelines)

## 1. 컬러 시스템 고도화 (AI 템플릿 느낌 제거)
- **메인 컬러(오렌지) 조정:** 다크 모드에서 시각적 진동을 유발하지 않도록 기존의 쨍한 오렌지색의 채도를 살짝 낮추고 명도를 올려 부드럽고 세련된 파스텔톤/네온톤 오렌지(예: `#FF8A4C` 등)로 변경해.
- **서브 컬러(배경 및 표면) 통일감:** 단순한 무채색(`#111`, `#222`) 대신, 배경과 카드 UI 컴포넌트의 짙은 회색 톤에 메인 오렌지 컬러의 틴트를 아주 미세하게(1~2%) 섞어 차가운 느낌을 없애고 전체적인 바이브를 통일해.

## 2. 시각적 대비와 접근성 (WCAG 2.1 기준 준수)
- **결과 카운트('7곳 찾음') 텍스트:** 현재 배경과 묻히는 짙은 회색을 밝은 회색(예: `text-gray-400` 또는 `text-gray-300` 수준의 색상)으로 올려 명도 대비(Contrast Ratio) 4.5:1 이상을 확보해.
- **외부 플랫폼 아이콘(네이버 'N' 등):** 쨍한 초록색이 서비스 무드를 해치고 있으므로, 흑백(Grayscale) 처리 후 투명도를 주거나, 전체 UI 톤앤매너에 맞게 시각적 충돌을 최소화해.

## 3. 게슈탈트 원칙에 따른 정보 그룹화 및 형태 분리
- **액션 칩 vs 정보 태그 분리:** 
  - 상단의 음식 카테고리(한식, 일식 등)는 '누를 수 있는 버튼'임을 명확히 하기 위해 둥근 알약 형태(Pill 모양의 강한 border-radius)를 유지하고 호버/액티브 상태의 시각적 텐션을 추가해. 
  - 반면, 하단 식당 카드의 태그(`#떡볶이` 등)는 '단순 정보'이므로 모서리 둥글기(Border-radius)를 덜어내어 약간 각지게 만들고, 폰트 크기와 웨이트를 낮춰 액티브 칩과 헷갈리지 않게 시각적으로 분리해.
- **설명 영역 보더 라인 개선:** 식당 카드 하단의 설명 텍스트("스트레스 풀리는...") 좌측에 있는 오렌지색 포인트 보더 라인이 텍스트와 겉돌지 않도록 해. 텍스트와 보더 사이의 패딩(Padding)을 자연스럽게 조정하고 배경에 아주 옅은 오렌지색(`#FF8A4C`의 5~10% opacity) 패널을 깔아 하나의 온전한 정보 블록으로 인지되게 묶어줘.
