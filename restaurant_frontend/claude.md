# PickEat 백엔드 구축 지시서 (for Claude Agent Team)

## 🎯 목표 (Objective)
현재 폴더(`restaurant_frontend/`)에 완성된 프리미엄 다크 모드 기반의 프론트엔드 UI를 위해, 실제 데이터를 연동하여 동작할 수 있는 **백엔드(Backend) 시스템**을 구축하고 팀 단위 모의 코드 리뷰까지 수행해 주세요.

---

## 🏗 팀 구성 및 역할 세분화 요구사항
너희는 단일 AI 모델이 아니라 '전문 에이전트 팀(Agent Team)'으로서 역할을 나누어 이 프로젝트를 수행해야 해:
1.  **Lead Backend Engineer:** 서버 아키텍처 설계, API 최적화.
2.  **API Data Specialist:** 구글/네이버 지도 데이터 연동 전략 수립.
3.  **Security & Code Reviewer:** 작성된 백엔드 코드의 보안 취약점을 검토하고 보완점 제시.

---

## 📋 핵심 요구사항 (Key Requirements)

### 1. 프론트엔드 연동을 위한 API 엔드포인트 설계
현재 `app.js`에 작성된 `mockRestaurants` 가짜 데이터를 대체할 수 있는 RESTful API를 설계해. 반환해야 하는 JSON 필수 속성은 다음과 같아:
- `id` (식당 고유번호)
- `name` (식당 이름)
- `category` (대분류 - 한식/일식 등)
- `sub` (배열 - 소분류 태그 및 키워드)
- `dist` (사용자 위치 기반 계산된 거리)
- `rating` (별점 평점)
- `price` (가격대 텍스트)
- `desc` (핵심 리뷰 및 설명 1~2줄 요약)
- `platforms` (배열 - 'google' 또는 'naver')
- `img` (식당 대표 사진 썸네일 URL)

### 2. 위치 및 로컬 데이터(지도 API) 처리 아키텍처
현재 웹앱은 반경(0.1km ~ 5.0km)으로 식당을 필터링해야 해. 이 구조를 위해 **어떤 언어 및 프레임워크(예: Node.js/Express, Python/FastAPI 등)**와 **어떤 데이터베이스 구조**를 추천할지, 그리고 구글 Places API 및 네이버 로컬 API 데이터를 어떻게 융합(Merge)하여 가져올지 설계 방향을 함께 제시해. 백엔드 구성 시 가장 경제적인 최적의 스택을 선택해서 코드를 작성해 줘.

### 3. 클라이언트(프론트엔드)-서버 통신 로직 추가
프론트엔드의 `app.js`에서 더미 데이터를 지우고, 네가 구축한 백엔드 API를 `fetch`를 이용해 실제 호출하는 비동기 함수 구조(ex. `fetchRestaurants(lat, lng, distance)`)로 `app.js`를 수정 혹은 확장안을 제시해.

### 4. 시큐리티 및 리뷰 (Team Review)
백엔드 코드를 짜고 나면, **에이전트 간의 리뷰(Code Review) 스크립트**를 작성해. 즉, 한 에이전트가 코드를 어떻게 최적화했는지 리포트하면, 다른 리뷰어 에이전트가 "API 호출 제한률(Rate Limit) 방어 로직이 더 필요하다", "사용자 위치 정보(Geolocation) 암호화가 빠졌다" 등의 피드백을 주고 적용한 최종 버전을 나에게 제출해.

## 🚦 실행 절차
1.  이 폴더의 `index.html`, `style.css`, `app.js`를 전체적으로 훑어보고 프론트 구조를 파악해.
2.  백엔드 코드(예: `server` 폴더 생성 후 초기화)를 작성하고,
3.  작성한 코드에 대한 **'팀 단위 리뷰 리포트'**를 함께 제출해줘. 시작해!
